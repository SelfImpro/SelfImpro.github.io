Js函数

在Javascript中定义一个函数，有两种写法：

  • function foo() { }
  • var foo = function () { } ， 建议使用的写法

两种写法完全等价。但是在解析的时候，前一种写法会被解析器自动提升到代码的头部，因此违背了函数应该先定义后使用的要求，所以建议定义函数时，全部采用后一种写法。

案例1： 以下是个人对js这种语法的一种理解。下面可以拆分为几段

  1. (function($){})(jQuery);这个会比较熟悉一点就是声明一个匿名函数然后调用，jQuery是传的参数，是Jquery框架的一个对象。
  2. $.extend({});表示继承$，而这里的$就是传进来的jQuery，然后实现一些自己的方法，方便直接通过$.function的形式调用
  3. {what: function(string){}} 在对象体中可以这样定义函数。var a = {what: function(){}}通过a.what(); 调用

说到这里有点想到js里面的对象和对象继承是个什么情况

!function(){}() 和 +function(){}()

直接在IDE里面写function (){}()，编译器会报错无法通过，如果我们在前面加上!或者+就不会了；也可以在外面包裹一个小括号，那就是上面的写法。这种函数都表示立即执行，没有其他用处。

prototype是个什么👻

参考文章

  1. JavaScript世界万物诞生记 这篇文章写的比较有意思，虽然对js对象、原型有些地方有点模糊了，但是整体还是不错的。

prototype是创建创建对象的原型，一个模板。如果想要继承他则需要复制一份模板，然后在模板的基础之上进行修改。这里复制的模板就赋值给__proto__了。这就是一个对象里面有一个constructor和__proto__两个属性了。

要说prototype，就得说说__proto__。prototype是函数的内置属性，proto是对象的内置属性, 其实他们指向的是同一个对象。

js里面叫做原型，每一个对象内部都和另外一个对象关联，“另一个”对象就是原型，同时每一个对象都从原型继承属性。也可以说是继承的父类。

获取对象原型可以通过Object.getPrototypeOf(obj)获取，有获取就有设置’Object.create(obj)’，由创建的时候传入。默认的对象获取到的原型都是Object实例对象。
